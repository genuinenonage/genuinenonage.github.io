第一章 Web概述

Java Web简介

使用JavaEE技术体系开发企业级互联网项目，与JavaSE阶段有很大区别。在互联网下需要有客户端和服务器的区分。

JavaWeb技术栈

前端部分

HTML CSS JavaScript ES6 Nodejs npm vite Vue3 router axios pinia element-plus …

后端部分

HTTP Xml Tomcat Servlet Request Response Cookie Session Filter Listener MySQL JDBC Druid JSON lombok JWT …

JavaWeb的CS和BS模式

CS模式 Client-Server

用户在客户端需要安装程序，服务器部署服务程序。

BS模式 Browser-Server

浏览器为客户端

前后端分离

非前后端分离

程序员需要同时编写前后端代码，压力较大。

后端如果使用动态页面技术（JSP，Thymeleaf等），必须部署在一起。

前后端分离

后端只需按照接口文档开发，无需编写前端代码，前后端压力降低。

前端使用前端的页面动态技术，通过Vue等框架，工程化项目，独立部署到服务器。

第二章HTML&CSS

简介

HTML&CSS&JavaScript的作用

HTML用于网页主体结构的搭建。

CSS用于页面元素美化。

JavaScript用于页面元素的动态处理。

什么是HTML

HTML是Hyper Text Markup Language的缩写。超文本标记语言。

超文本：HTML本质上是文本文件，而普通的文本文件只能显示字符。但超文本能呈现出来的效果超过文档（例如图片、音频、视频、其他网页引入到当前HTML网页中）。

标记语言：不像Java这样的编程语言，由一系列标签组成的，每个标签都有固定的含义。

无序列表，每一项前面的符号不带有顺序特征。列表标签ul，列表项标签li。

特殊字符

有特殊含义的字符，需要同多转义字符来表示。

CSS的使用

CSS层叠样式表，能够对网页中元素位置的排班进行像素级控制，支持几乎所有的字体字号样式。简单来说，美化页面。

css引入方式

行内式：通过元素开始标签的style属性引入。

内嵌式：在head标签中,通过一对style标签定义CSS样式

外部样式表：在head标签中,通过link标签引入外部CSS样式

css选择器

元素选择器：根据标签名确定样式的作用范围，语法为 元素名 {}

id选择器：根据元素id属性的值确定样式的作用范围，语法为 #id值 {}

class选择器：根据元素class属性的值确定样式的作用范围，语法为 .class值 {}

css浮动

CSS的Float（浮动）使元素脱离文档流，按照指定的方向移动（左或右发生移动），直到它的外边缘碰到包含框或另一个浮动框的边框为止。

浮动的样式名：float，值：left向左浮动，right向右浮动，none默认值元素不浮动。

CSS定位

指定元素的定位类型position。

静态定位：static，默认值。

绝对定位：absolute，元素在页面上的固定位置。当前元素会让出原来的位置，其他元素可以占用。

相对定位：relative，相对于自己原来的位置进行定位。

固定定位：fixed，在浏览器窗口固定位置，不会随和页面的上下移动而移动。

CSS盒子模型

所有HTML元素可以看作盒子。可以设置HTML元素周围的内容。

边距（margin），边框（border），填充（padding），和实际内容（content）。

第三章JavaScript

曾经交JavaScript，后来被ECMA权威工会倡导所有浏览器厂商兼容，改名为ECMAScript，简称ES。

Vue3必须需ES6来支持。

JS特点

脚本语言：JS是一种解释型的脚本语言。不同于Java等语言先编译后执行，JS不会产生编译的字节码文件，而是直接在程序运行过程中对源文件进行解释。

弱类型：声明一个变量可以接受任何类型的对象。并且会在程序执行过程中根据上下文自动转换类型。

跨平台性：JS不依赖操作系统，仅需要浏览器支持。

什么是BOM编程

BOM是Browser Object Model的简写，即浏览器对象模型。有一系列对象组成，可以访问、控制、修改浏览器的属性和方法。

BOM编程的对象结构如下

  • window 顶级对象,代表整个浏览器窗口
    • location对象 window对象的属性之一,代表浏览器的地址栏
    • history对象 window对象的属性之一,代表浏览器的访问历史
    • screen对象 window对象的属性之一,代表屏幕
    • navigator对象 window对象的属性之一,代表浏览器软件本身
    • document对象 window对象的属性之一,代表浏览器窗口目前解析的html文档

什么是DOM编程

使用document对象的API完成对网页HTML文档进行动态修改,以实现网页数据和样式动态变化效果的编程。

JS的引入方式

内部脚本方式引入，在页面中,通过一对script标签引入JS代码。

外部脚本方式引入，通过script标签引入外部脚本文件。

表单事件

通过属性绑定。

通过DOM编程绑定。

事件的触发

行为触发。

会话级存储和持久级存储

通过BOM编程实现

会话级存储：浏览器在内存上临时存储的数据，浏览器关闭后，数据失去，通过window的sessionStorge属性实现。

持久级存储：浏览器在磁盘上持久存储的数据，浏览器关闭后，数据仍在，通过window的localStorge实现。

第四章 web后端服务

XML

是EXtensible Markup Language的缩写，可扩展标记语言。XML和HTML一样都是标记语言，它们的基本语法都是标签。

可扩展：XML允许自定义格式。但这不代表可以随便写。

可以使用DOM4J进行XML文件解析。

Tomcat

当前应用最广的JavaWeb服务器。Web服务器通常由硬件和软件共同构成。

Web项目的标准结构

  • static非必要目录，一般在此处放静态资源（CSS JS IMG）。
  • WEB-INF 必要目录，受保护的资源目录，浏览器通过url不可以直接访问。
    • classes 必要目录，src下源代码、配置文件，编译后会在该目录下。
    • lib 必要目录，项目以来的jar编译后会在该目录。
    • web.xml 必要文件，web项目的基本配置文件。
  • index.html 非必要文件，index.html/index.htm/index.jsp为默认的欢迎页

Servlet

Servlet (server applet) 是运行在服务端(tomcat)的Java小程序，是sun公司提供一套定义动态资源规范; 从代码层面上来讲Servlet就是一个接口。

Servlet生命周期

（可以编码的阶段）

  1. 构造器对象
  2. 初始化
  3. 服务处理：仅有多次执行
  4. 销毁

Servlet继承结构

Servlet接口：所有Servlet必须实现接口中的定义的方法。

GenericServlect抽象类：对Servlet接口一些固定功能的粗糙实现,以及对service方法的再次抽象声明,并定义了一些其他相关功能方法。

HttpServlet抽象类：除了基本的实现以外,增加了更多的基础功能。

我们可以自定义实现Servlet的所有方法，也可使用抽象类，专注于业务的开发。

ServletConfig和ServletContext

ServletContext服务共享是单例的，ServletConfig是多例的，每个Servlet单独一个。

ServletConfig为Servlet提供初始配置参数的一种对象。

ServletContext可以为所有的Servlet提供初始配置参数。

请求转发和重定向

Tomcat会将请求报文转换为ServletRequest对象，会将ServletResponse对象转换相应报文。

转发操作的是ServletRequest
重定向操作的是ServlectReponse

乱码问题

根本原因：数据的编码和解码使用的不是同一个字符集，或者使用了不支持某个语言文字的字符集。

MVC架构模式

MVC（Model View Controller）是软件工程中的一种软件架构模式，它把软件系统分为模型、视图和控制器三个基本部分。用一种业务逻辑、数据、界面显示分离的方法组织代码。

开发建议

善于编写接口文档。
在接口层编写详细文档。

主动打印的异常，info。
非正常打印的异常，error。

三大域对象

请求域对象HttpServletRequest，一次请求之内及请求转发。

会话域对象HttpSession，一次会话之内，可以跨多个请求。

应用域对象ServletContext，本应用之内，可以跨多个会话。

过滤器

Filter不仅可以对请求做出过滤，还可以对相应再次进行处理。

监听器

监听域对象发生的事。

仅监听域对象的操作。监听应用域、会话域、请求域的创建、销毁、设置属性等事件。

Vite+Vue3项目的目录结构

什么是.vue文件？

传统的页面有.htm .css .js三个文件组成（多文件组件）

vue将这文件合并成一个.vue文件（Single-File Component，SFC，单文件组件），每个.vue文件可以理解为一个组件，多个.vue文件可以构成一个整体页面。

template标签 代表html部分代码

script标签 代表js代码

style标签 代表css代码

工程化vue项目如何组织这些组件？

  • index.html是项目的入口,其中 <div id ='app'></div>是用于挂载所有组建的元素
  • index.html中的script标签引入了一个main.js文件,具体的挂载过程在main.js中执行
  • main.js是vue工程中非常重要的文件,他决定这项目使用哪些依赖,导入的第一个组件
  • App.vue是vue中的核心组件,所有的其他组件都要通过该组件进行导入,该组件通过路由可以控制页面的切换

Vue3视图渲染技术

插值表达式

最基本的数据绑定形式是文本插值。语法双大括号{{}}。

插值表达式不绝对依赖标签,其位置相对自由

文本渲染

为了渲染双标中的文本,我们也可以选择使用v-text和v-html命令

v-text可以将数据渲染成双标签中间的文本,但是不识别html元素结构的文本

v-html可以将数据渲染成双标签中间的文本,识别html元素结构的文本

属性渲染

想要渲染一个元素的 attribute，应该使用 v-bind指令

事件绑定

使用 v-on 来监听 DOM 事件，并在事件触发时执行对应的 Vue的JavaScript代码。

用法：v-on:click="handler" 或简写为 @click="handler"

响应式基础

关键字ref

ref 可以将一个基本类型的数据（如字符串，数字等）转换为一个响应式对象。 ref 只能包裹单一元素。

关键字reactive

使用 reactive() 函数创建一个响应式对象或数组。

条件渲染

  • v-if='表达式' 只会在指令的表达式返回真值时才被渲染
  • 一个 v-else 元素必须跟在一个 v-if 元素后面，否则它将不会被识别。

v-show也是条件渲染，v-show 仅切换了该元素上名为 display 的 CSS 属性。

列表渲染

v-for 指令基于一个数组来渲染一个列表

双向绑定

v-model专门用于双向绑定表单标签的value属性,语法为 v-model:value='',可以简写为 v-model=''。

属性计算

computed() 方法期望接收一个 getter 函数，返回值为一个计算属性 ref。普通函数多次调用会执行多次，属性计算，属性不变时，多次调用只会调用一次。

数据监听器

使用 watch 函数在每次响应式状态发生变化时触发回调函数：

  • 当数据发生变化时需要执行相应的操作
  • 监听数据变化，当满足一定条件时触发相应操作
  • 在异步操作前或操作后需要执行相应的操作

watchEffect默认监听所有的响应式数据

Vue生命周期

常见钩子函数

  • onMounted() 注册一个回调函数，在组件挂载完成后执行。
  • onUpdated() 注册一个回调函数，在组件因为响应式状态变更而更新其 DOM 树之后调用。
  • onUnmounted() 注册一个回调函数，在组件实例被卸载之后调用。
  • onBeforeMount() 注册一个钩子，在组件被挂载之前被调用。
  • onBeforeUpdate() 注册一个钩子，在组件即将因为响应式状态变更而更新其 DOM 树之前调用。
  • onBeforeUnmount() 注册一个钩子，在组件实例被卸载之前调用。

什么是路由？

根据不同的URL地址展示不同的内容或页面。

路由的作用

  • 单页应用程序中，路由可以实现不同视图之间的无刷新切换，提升用户体验。
  • 可以实现页面的认证和权限控制。
  • 可以利用浏览器的前进与后退，帮助用户回到之前访问过的页面。

路由守卫

用于在路由切换期间进行一些特定任务的回调函数。

可以用于例如验证用户是否已登录、在路由切换前提供确认提示、请求数据等

  1. 全局前置守卫：在路由切换前被调用，可以用于验证用户是否已登录、中断导航、请求数据等。
  2. 全局后置守卫：在路由切换之后被调用，可以用于处理数据、操作 DOM 、记录日志等。
  3. 守卫代码的位置: 在router.js中

回调函数Promise

Promise对象有三种状态：Pending进行中、Resolved已完成、Rejected已失败。

基本用法

catch

Promise.prototype.catch方法是.then(null, rejection)的别名，用于指定发生错误时的回调函数。

async和await使用

async标识函数后,async函数的返回值会变成一个promise对象。如果函数内部返回的数据是一个非promise对象,async函数的结果会返回一个成功状态 promise对象。

await会等右边的promise对象执行结束,然后再获取结果,后续代码也会等待await的执行

await必须在async函数中，如果await右边的promise失败了,就会抛出异常,需要通过 try … catch捕获处理

跨域

浏览器同源禁止策略。CROS，前端处理或者后端处理，一方处理就可以了。

同源策略会阻止一个域的javascript脚本和另外一个域的内容进行交互。所谓同源（即指在同一个域）就是两个页面具有相同的协议（protocol），主机（host）和端口号

  1. 前端服务器配置代理，请求先通过前端来请求后端。
  2. 后端给预检请求相应允许跨域，设置时间限制，在这个时间范围内，允许跨域。

pinia详细

State (状态) 在大多数情况下，state 都是你的 store 的核心。人们通常会先定义能代表他们 APP 的 state。在 Pinia 中，state 被定义为一个返回初始状态的函数。

Getter 完全等同于 store 的 state 的计算值。可以通过 defineStore() 中的 getters 属性来定义它们。推荐使用箭头函数，并且它将接收 state 作为第一个参数：

Action 相当于组件中的 method。它们可以通过 defineStore() 中的 actions 属性来定义，并且它们也是定义业务逻辑的完美选择。类似 getter，action 也可通过 this 访问整个 store 实例，并支持**完整的类型标注(以及自动补全)**。不同的是，action 可以是异步的，你可以在它们里面 await 调用任何 API，以及其他 action！
