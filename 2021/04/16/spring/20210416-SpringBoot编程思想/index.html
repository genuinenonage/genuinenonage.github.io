总览Spring Boot

Spring Framework 时代

Spring Framework 一定程度上颠覆了J2EE开发模式，让我们知道除了J2EE标准外，还能用其他的模式开发。这依赖于Spring Framework的新概念：IOC（Inversion of Control，控制反转）和DI（Dependency Inject，依赖注入）。

J2EE与Spring Framework都与JSR（Java Specification Requests，Java标准规范）密不可分。

在J2EE技术体系中，IOC的实现方式为JNDI（Java Naming and Directory Interface，Java命名和目录接口），而DI是EJB容器注入。

无论Spring Framework使接口如何优化，由于自身并非容器，基本上必须随Java EE容器启动而装载。Spring Boot的出现，改变了Spring技术体系的现状。

Spring Boot 简介

Spring Boot 被官方定义为“BUILD ANYTHING”。

Spring Boot 为了快速启用最小化配置的Spring 应用而设计，并具有一套固定的视图，用于构建生产级别的应用。

Spring Boot 使构架独立的Spring应用变得简单，通过固定的视图来固化Spring Boot应用的依赖的Spring平台和第三方类库（只需要确定Spring Boot的版本，不用考虑Spring Framework和第三方类库的版本），并且大多数的Spring boot应用只需要少量的Spring配置（包括属性配置与代码配置）。

Spring Boot 特性

  1. 创建独立的Spring应用。
  2. 嵌入Tomcat、Jetty或Undertow等Web容器（不需要部署WAR文件）。
  3. 提供固化的“starter”依赖，简化构建配置。
  4. 当条件满足时自动装配Spring或第三方类库。
  5. 提供运维（Production-Ready）特性，如指标信息、健康检查及外部化配置。
  6. 无代码生成，不需要xml配置。

理解独立的Spring应用

开发人员常将Spring Boot框架构建的应用称为Spring Boot应用，实际上是Spring应用。

在大多数Spring Boot应用场景中，程序直接或间接地使用SpringApplication API引导应用，结合嵌入式Web容器，对外提供HTTP服务。

当Spring Boot 出现嵌入式容器启动方式后，嵌入式容器成为Spring容器上下文中的组件Beans，随Spring应用上下文启动而注册并初始化。驱动Spring应用上下文启动的核心组件是Spring Boot核心API SpringApplication，所以是Spring应用，也可以称为Spring Boot应用。

理解嵌入式Web容器

Spring Boot可以嵌入Servlet Web容器和Reactive Web 容器。统称嵌入式Web容器，容器之间是互斥关系，无法并存。

嵌入式Servlet Web容器

Spring Boot 2.0支持三种嵌入式**Servlet 3.1+**容器。

包括：Tomcat、Jetty、Undertow
